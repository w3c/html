HTML 5.1 Implementation report

This document outlines known implementations for the features added between HTML 5.0 and HTML 5.1. Note that there were a number of changes that removed poorly-interoperable features.

Documented changes

For the following changes, implementation of the feature is documented, either in the pull request or related issue, or through documentation for developers.

Only one known implementation

:dir CSS selector matching

reportValidity() method on HTMLInputElement interface

Events and APIs for tracking promise rejection
This feature appears only to be implemented in blink.

Apparently unimplemented

probablySupportsContext() for canvas
probablySupportsContext() is a convenience function meant to return false if called on a canvas that definitely doesn't support the relevant context, and true otherwise.
